# Contributing to KCS Meet

## 🙏 Divine Contribution Guidelines

We welcome spiritual contributions from all devotees and developers! Your participation helps us spread Krishna's universal vision through technology.

## 🌸 How to Contribute

### Prerequisites
- 🧘 Node.js (v18+ Enlightenment)
- 🌿 npm or Yarn (Path of Dependency)
- 🕯️ Git (Version Control Wisdom)
- 💖 Pure intention to serve

### Setting Up Development Environment

1. **Fork and Clone the Repository**
   ```bash
   git clone https://github.com/YOUR_USERNAME/KCS-App.git
   cd KCS-App
   ```

2. **Install Dependencies**
   ```bash
   npm install
   ```

3. **Create Environment File**
   ```bash
   cp .env.example .env.local
   # Fill in your sacred credentials
   ```

4. **Start Development Server**
   ```bash
   npm run dev
   ```

## 🌼 Types of Contributions

### 🔧 Bug Fixes
- Report bugs with detailed descriptions
- Include steps to reproduce
- Provide browser/device information
- Submit PRs with minimal, focused fixes

### ✨ Feature Enhancements
- Discuss new features in issues first
- Ensure features align with spiritual values
- Include proper documentation
- Write tests when applicable

### 📚 Documentation
- Improve clarity and completeness
- Add missing API documentation
- Update installation guides
- Translate content (future scope)

### 🎨 UI/UX Improvements
- Follow existing design patterns
- Ensure mobile responsiveness
- Maintain accessibility standards
- Test across different devices

## 🌟 Development Workflow

1. **Create Feature Branch**
   ```bash
   git checkout -b feature/divine-enhancement
   ```

2. **Make Changes**
   - Write clean, readable code
   - Follow existing code style
   - Add appropriate comments
   - Test your changes

3. **Commit with Meaningful Messages**
   ```bash
   git commit -m "feat: add transcendental video filters"
   ```

4. **Push and Create PR**
   ```bash
   git push origin feature/divine-enhancement
   ```

## 🕉️ Code Standards

### TypeScript Guidelines
- Use TypeScript for all new files
- Provide proper type definitions
- Avoid `any` type when possible
- Use meaningful variable names

### React Best Practices
- Use functional components with hooks
- Implement proper error boundaries
- Optimize re-renders with useMemo/useCallback
- Follow component composition patterns

### Styling Guidelines
- Use Tailwind CSS utility classes
- Follow mobile-first approach
- Maintain consistent spacing
- Ensure dark mode compatibility
